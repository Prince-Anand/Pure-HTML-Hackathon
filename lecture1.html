How does Internet Work?

Before Moving to How, Why do we need Internet?
To transfer data from one computer to another, a direct electronic connection is required. The Internet is the largest such network that connects computers worldwide.
Now, let's see how does one Computer know where to send the data through IP Addresses.

IP Address

Internet Protocol(IP) Address is just like address of a computer on a network to share data.
Since, Computer understands only 0's and 1's, IP address is a collection of bits.

There are two types of IP Address:

  • IPv4 - 32 bit address (represented in 4 decimal octets)
  • IPv6 - 128 bit address (represented in 8 groups of hexadecimal)
               Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Message passing through IP Address
         

IPv6 is still not widely used. So, For Billions of devices all over the internet, IPv4 32 bit is not sufficient.
Therefore, we use Public and Private IP Address.

  • Public IP Address - Unique across the whole internet. Assigned by ISP(Internet Service Provider)
    For example, your home network has a public IP address assigned by your ISP like Airtel, Jio etc.
    This can sometimes change if you restart your router. It is reassigned by ISP.

  • Private IP Address - Unique within a private/local network. Assigned by Router
    For example, in a home network, all devices connected to the router have a private IP address assigned by the router.
    Private IP Address ranges(reserved):
    • 10.0.0.0 to 10.255.255.255 (Used by large corporations)
    • 172.16.0.0 to 172.31.255.255 (Used by medium-sized networks)
    • 192.168.0.0 to 192.168.255.255 (Most common for home networks)

Two devices on the internet at different locations can have same private address. But, they will have different public IP

Now, to make it easy for humans to remember IP Address, we use Domain Names.
For example, instead of remembering digits like 142.250.72.206, we can remember google.com.

Domain Name System(DNS)

DNS is a hierarchical and distributed naming system that translates domain names into IP addresses.
Jab Hum kisi Website ka naam likhte hai toh DNS us naam ko IP address me convert karta hai jisse ki computer samajh sake.
For Example, jab aap apne browser me google.com likhte hai toh wo DNS server se contact karta hai jisse ki uska IP address mil sake.
DNS servers are like phone books for the internet. They maintain a directory of domain names and translate them to IP addresses.

How DNS Works? E.g, If we type google.com

  1. Local Check (Ghar ki Talashi) - Browser pehle apne cache me check karta hai ki kya uske paas google.com ka IP address already stored hai. Agar hai toh wo directly us IP address pe chala jata hai.

  2. Recursive DNS Resolver (Padosi se poochna) - Agar browser ke paas IP address nahi hai toh wo apne configured recursive DNS server se poochta hai. Ye server usually aapke ISP(Internet Service Provider) ke dwara provide kiya jata hai.

  3. Asli Khoj (The Real Search) - Agar DNS Resolver ke cache memory me use haal hi me google.com ka koi search nhi hua hai toh wo asli khoj shuru karta hai. Ye process me 4 types ke DNS servers involved hote hai:

    • Root DNS Servers - Ye servers internet ke top level pe hote hai. Ye top-level domain(TLD) servers ke baare me jankari rakhte hai. Jaise ki .com, .org, .net etc.
      E.g., Yha Resolver puchta hai bhai, .com domain ki jaankari kahan milegi?

    • TLD DNS Servers - Ye servers specific top-level domains ke liye responsible hote hai. Jaise ki .com TLD server google.com ke authoritative DNS server ke baare me jankari rakhte hai.
      E.g., Yha Resolver puchta hai bhai, google.com domain ki jaankari kahan milegi?

    • Authoritative DNS Servers - Ye servers specific domain names ke liye IP addresses rakhte hai. Jaise ki google.com ka authoritative DNS server uska actual IP address rakhte hai.
      E.g., Yha Resolver puchta hai Finally bata bhai, google.com ka IP Address aakhir hai kya?

  4. The Response - Authoritative server se IP address milne ke baad, DNS Resolver us jawab (142.250.196.110) ko aapke computer ko bhej deta hai. Saath hi, woh is jawab ko apni cache mein save kar leta hai taaki agli baar itni lambi khoj na karni pade.

  5. Connection Established - Ab aapka browser is IP address (142.250.196.110) ka use karke google ke server se connect ho jata hai aur website load ho jati hai.

Mac Address

MAC (Media Access Control) Address is a unique identifier assigned to a network interface controller (NIC) for use as a network address in communications within a network segment.
It is a 48-bit address, usually represented in hexadecimal format, e.g., 00:1A:2B:3C:4D:5E.
MAC Address is used for communication within a local network segment. It is used by switches to forward data packets to the correct device on the network.
Since IP Address can change sometimes, MAC Address is used to uniquely identify a device on a network.
Suppose Agr hum message bhejte hai aur usi time hamara IP address change ho jata hai toh hmara message kisi galat device pe chala jayega.
Isliye, MAC address ka use kiya jata hai to ensure that the message reaches the correct device.

Port Number

Ab Agar ek hi device pe multiple applications run kar rahe hai, toh kaise pata chalega ki kaunsa data kis application ke liye hai?
Port Number is a numerical identifier in networking used to specify a particular process or service on a device.
It is a 16-bit number, ranging from 0 to 65535.
For example, when you access a website, your browser connects to the web server using IP address and port number 80 (HTTP) or 443 (HTTPS).
Common Port Numbers:

  • HTTP - 80
  • HTTPS (secured websites) - 443
  • FTP - 21
  • SSH - 22
  • DNS - 53

LAN, Switch and Router

  • LAN (Local Area Network) - A LAN is a network that connects devices within a limited area, such as a home, office, or building.
    It allows devices to communicate and share resources like files, printers, and internet connections.
  • Switch - Switch ek device hai jo multiple computers/devices ko connect karke unke beech data forward karta hai MAC address ke base par.
  • Router - Router ek device hai jo multiple networks ko connect karta hai aur data packets ko unke destination IP address ke base par forward karta hai.

What is Web Development? - Client-Server Model

Internet pe hum jo bhi website visit karte hai wo Client-Server Model pe kaam karta hai.
To make these websites is the role of a Web Developer. Client-Server Model ek architecture hai jisme client aur server ke beech communication hota hai.
Client wo device hai jo request bhejta hai aur server wo device hai jo request ko We write code to create these websites using HTML, CSS, JavaScript for frontend and various backend technologies like Node.js, Python, PHP etc.

Mern Stack

Mern Stack is a popular web development stack that consists of four main technologies: MongoDB, Express.js, React.js, and Node.js.

  • MongoDB is used as the database
  • Express.js is used as the backend framework
  • React.js is used as the frontend framework
  • Node.js is used as the runtime environment
