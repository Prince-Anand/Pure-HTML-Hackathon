File Path in HTML

File path is used to link one file to another file. There are two types of file paths:

  • Absolute Path: It is the complete path from the root directory to the target file.It starts with a drive letter (e.g., C:\) on Windows or a forward slash (/) on Unix-based systems.
    Mtlb, agr hame apne pc me present kisi file ko browser ko btana h toh hme uska full path (e.g. C:\Users\Prince\Desktop\Web Development - Defense\project\blogs.html) dena hoga
    Ye hum locally apne pc pr kr skte h but this is not allowed in browser when the website is hosted

  • Relative Path: It is the path relative to the current file's location. It does not start with a drive letter or a forward slash. Instead, it uses dots (.) and slashes (/) to navigate through directories.
    • ./ - Current Directory
    • ../ - Parent Directory (One level up)
    • ../../ - Two levels up
    • And so on...

Problem with Absolute Path:

  • Agar hum apne website me absolute path use krte h toh jab hum apne website ko kisi server pr host krenge toh wo file us server pr exist nhi kregi. Isliye hamesha relative path use krna chahiye.

  • Browsers have security restrictions that prevent web pages from accessing local files on a user's computer. This is to protect user privacy and security.
    Agar ye rule na ho toh koi bhi malicious website hamare pc ki files ko access kr skti h without our permission.

Therefore, Always use Relative Path in your HTML files . Absolute should be avoided except for linking to external resources like CDNs (Content Delivery Networks) or external websites.

HTML BoilerPlate Code

This is the code that we write every time we create a new HTML file. It is the basic structure of an HTML document.
It includes:

  • <!DOCTYPE html>: This declaration defines the document type and version of HTML being used. It helps browsers to render the page correctly.

  • <html lang="en">: This is the root element of the HTML document. The lang attribute specifies the language of the document, which is important for accessibility and SEO.

  • <head>: This section contains meta-information about the document, such as the character set, viewport settings, title, and links to external resources like CSS files or JavaScript files.

  • <meta charset="UTF-8">: This meta tag specifies the character encoding for the document, ensuring that it can display a wide range of characters correctly.
    For example, characters from different languages, special symbols, and emojis. like ASCII (English characters) only support a limited set of characters, while UTF-8 can represent virtually all characters used in modern computing even emojis like ðŸ’–, ðŸ¤£, etc;

  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: This meta tag is crucial for responsive web design. It sets the viewport to match the device's width and initial zoom level, ensuring that the page looks good on all devices, including mobile phones and tablets.

  • <title>Document</title>: This tag sets the title of the webpage, which is displayed in the browser tab and used by search engines.

  • <body>: This section contains the main content of the webpage that is visible to users, including text, images, links, and other elements.

DIV, Class, and ID in HTML

Everything in HTML is an element. All HTML elements can be classified into two categories:
  • Block-level elements - Takes Full Width of page
  • Inline elements - Takes Width according to content present.
All Elements are percieved as boxes by the browser.

One such box is div(division) tag.

DIV: The <div> tag is a block-level container used to group together HTML elements. It is commonly used for layout purposes and to apply styles to a section of a webpage.

Website me bhot sare elements ek jaise ho skte h (jese ki multiple paragraphs, multiple images, etc), even Multiple div tags bhi ho skte h.
Toh agar hme un sabhi elements ko ek saath style krna h toh hme un sabhi elements ko alag alag style krna pdega.

Is problem ko solve krne ke liye hum Class and ID ka use krte h.

  • Class: The class attribute is used to assign a class name to an HTML element. Multiple elements can share the same class name, allowing you to apply the same styles to all of them.
    Class names should be descriptive and meaningful, and they should not contain spaces. If you want to use multiple classes for an element, you can separate them with spaces.
    Example: <div class="container highlight">

  • ID: The id attribute is used to assign a unique identifier to an HTML element. Each id must be unique within a document, meaning that no two elements can have the same id. IDs are often used for targeting specific elements with CSS or JavaScript.
    Example: <div id="header">
